NonOI support functions. 

Polls the serial input for data belonging to a sensor data stream previously requested with \hyperlink{class_arduino_roomba_a687c72eb69928dba45aa53aa39af2f1f}{stream()}. As sensor data is read it is appended to dest until at most len bytes are stored there. When a complete sensor stream has been read with a correct checksum, returns true. See the Open Interface manual for details on how the sensor data will be encoded in dest. Discards characters that are not part of a stream, such as the messages the Roomba sends at startup and while charging. Create only. No equivalent on Roomba. 
\begin{DoxyParams}[1]{Parameters}
\mbox{\tt out}  & {\em dest} & Destination where the read data is stored. Must have at least len bytes available. \\
\hline
\mbox{\tt in}  & {\em len} & Max number of sensor data bytes to store to dest \\
\hline
\end{DoxyParams}
\begin{DoxyReturn}{Returns}
true when a complete stream has been read, and the checksum is correct. The sensor data (at most len bytes) will have been stored into dest, ready for the caller to decode. 
\end{DoxyReturn}

Starts the Open Interface and sets the mode to Passive. You must send this before sending any other commands. Initialises the serial port to the baud rate given in the constructor

The iRobot Roomba and Create platforms support a serial port through which you can control and interrogate the device. The protocol implemented here conforms to the Open Interface protocol described in the iRobot Open Interface Command Reference. Not all commands are supported on both platforms. Differences are noted in the API

The Roomba and Create is equipped with a min-\/din serial port socket, while the Create is also equipped with a 25-\/pin D connector called the Cargo Bay Connector. The pins on the Cargo Bay Connector include the serial port, battery, digital inputs and outputs, and an analog input.

In order to communicate with a Roomba, you must create an instance of the Roomba class and then call its instance methods to send commmands and receive sensor messages. You can also request continuous streams of sensor data to be sent by the Roomba. The Roomba also emits messages on its serial port from time to time as described below. \hypertarget{class_arduino_roomba_a687c72eb69928dba45aa53aa39af2f1f}{
